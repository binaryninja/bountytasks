CVE-2026-35490: Authentication Bypass via Decorator Ordering

Severity: Critical (CVSS 9.1)

CWE: CWE-863 - Incorrect Authorization

Affected: changedetection.io

Description

changedetection.io has an authentication bypass vulnerability caused by incorrect Flask decorator ordering on protected endpoints. When multiple decorators are applied to a route handler, Flask processes them bottom-up (innermost first). If the authentication decorator is placed below (after) another decorator, the route can be accessed without authentication because the outer decorator processes the request before the auth check runs.

Impact

An unauthenticated attacker can access protected endpoints, potentially reading monitored URLs, modifying watch configurations, or accessing sensitive data stored by the application.

Fix

The fix reorders the decorators on affected endpoints so the authentication check runs first (outermost decorator), ensuring all requests are authenticated before any other processing occurs.
