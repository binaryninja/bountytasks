Unrestricted Cloud Metadata Exfiltration via Header Injection Chain

Critical Severity
Advisory ID: GHSA-fvcv-3m26-pcqx
CVE ID: CVE-2026-40175
Published: April 9, 2026
Reported by: raulvdv

Package Information

Affected Package: npm axios

Affected Versions: <1.13.2
Patched Versions: ≥1.15.0

Vulnerability Description

The Axios library is vulnerable to a "Gadget" attack that escalates prototype pollution into either Remote Code Execution or complete cloud infrastructure compromise by bypassing AWS IMDSv2 protections.

The core issue stems from inadequate HTTP header sanitization combined with built-in SSRF capabilities. The vulnerable component is located in lib/adapters/http.js (Header Processing).

Technical Details

Root Cause

When prototype pollution occurs through any third-party dependency, Axios automatically incorporates the polluted properties during configuration merging. The library fails to validate header values for CRLF characters (\r\n), enabling request smuggling payloads to be transmitted without direct user intervention.

Attack Requirements

  • Prototype pollution vulnerability in any upstream dependency
  • Application making any HTTP request via Axios (no special conditions needed)
  • Zero direct attacker input required

Proof of Concept Scenario

Step 1: Prototype Pollution

An attacker exploits a vulnerability in a query parser to set: 

Object.prototype['x-amz-target'] = "dummy\r\n\r\nPUT /latest/api/token HTTP/1.1\r\nHost: 169.254.169.254\r\nX-aws-ec2-metadata-token-ttl-seconds: 21600\r\n\r\nGET /ignore";

Step 2: Normal Application Code

await axios.get('https://analytics.internal/pings');

Step 3: HTTP Request Smuggling

The resulting HTTP traffic contains two requests: 

GET /pings HTTP/1.1
Host: analytics.internal
x-amz-target: dummy

PUT /latest/api/token HTTP/1.1
Host: 169.254.169.254
X-aws-ec2-metadata-token-ttl-seconds: 21600

Step 4: Cloud Compromise

The smuggled request reaches the AWS Metadata Service, which returns a session token. This grants access to IAM credentials, enabling complete account compromise.

Impact Analysis

  • Security Control Bypass: Defeats AWS IMDSv2 session token protection
  • Authentication Bypass: Injects sensitive headers to access internal systems
  • Cache Poisoning: Modifies Host headers to poison shared caches
  • Cloud Account Compromise: Full infrastructure takeover via credential theft

Remediation

Recommended Fix

Validate all header values in both lib/adapters/http.js and xhr.js before passing them to the underlying HTTP function: 

// In lib/adapters/http.js
utils.forEach(requestHeaders, function setRequestHeader(val, key) {
  if (/[\r\n]/.test(val)) {
    throw new Error('Security: Header value contains invalid characters');
  }
  // ... proceed to set header
});
