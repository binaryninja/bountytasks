Description

The simple-git library includes a security mechanism (blockUnsafeOperationsPlugin) that attempts to block dangerous Git options like -u and --upload-pack to prevent arbitrary command execution. This was originally introduced to address CVE-2022-25860.

However, the implementation fails to account for Git's flexible option-parsing behavior. Git allows combining single-character flags into a single argument (e.g., -vu is equivalent to -v -u). The safety check only matches arguments that start exactly with -u or --upload-pack, missing combined flags such as:

  • -vu (verbose + upload-pack)
  • -4u (IPv4 + upload-pack)
  • -6u (IPv6 + upload-pack)
  • -lu (local + upload-pack)
  • -nu (no-checkout + upload-pack)
  • -qu (quiet + upload-pack)
  • -su (shallow-submodules + upload-pack)
  • --u (long-form prefix match for upload-pack)

Impact

An attacker who can control arguments passed to git.clone(), git.fetch(), git.pull(), or git.push() can achieve arbitrary command execution on the host system, even when allowUnsafePack is set to false (the default).

Fix

The fix (released in simple-git 3.32.0) replaces the simple string-matching approach with a proper argument parser (@simple-git/argv-parser) that correctly identifies unsafe options regardless of how they are combined or formatted in the command line arguments.
