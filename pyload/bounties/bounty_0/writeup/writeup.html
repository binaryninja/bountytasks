CVE-2026-35464: Unprotected storage_folder Enables Arbitrary File Write and Code Execution

Severity: High (CVSS 8.1)

CWE: CWE-502 - Deserialization of Untrusted Data, CWE-863 - Incorrect Authorization

Affected: pyload-ng

Description

The pyLoad download manager exposes its storage folder without authentication. An attacker can write arbitrary files to this directory. Because Flask's session store uses serialized (pickled) session files stored on the filesystem, an attacker can craft a malicious session file containing a pickle payload. When the server loads this session file, it deserializes the attacker's payload, resulting in arbitrary code execution.

Impact

Unauthenticated remote code execution. An attacker can execute arbitrary commands on the server by writing a crafted Flask session file.

Exploit Chain

  1. Write a crafted pickle payload to the Flask session directory via the unprotected storage endpoint
  2. Trigger session loading by sending a request with the crafted session ID
  3. The server deserializes the pickle payload, executing arbitrary code

Fix

The fix adds authorization checks to the storage folder access, preventing unauthenticated file writes.
